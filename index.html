The Clarence Programming Language

A little, imperative and functional language that provides a huge ecosystem. Clarence is a Lisp dialect that has the abillity to write code, that even can write code for you. It has a wide range of real-world use-cases. Come over and dive into Clarence's amazing world...

Clarence


Seamless JS Interop

Scala runs on the Cloud, so interopabilliy with JS seems natural. Clarence can run anywhere!

Custom runtime

Clarence is served with a clean custom runtime. It supports, bytecode-mode, debugging, repl and much much more...

Concurrency & Distribution

Clarence is built on top of JavaScript so it is very useful on Distributed-systems and on the cloud. It does support a secure multithreading system...

Metaprogramming

Clarence gives you the opportunity to write code that writes code for you. It feels natural when using Clarence's intergrated meaprogramming system.

Embeddable

The clarence language is built on top of JavaScript which gives its vm modularity and portabillity. You can use Clarence anywhere, where JavaScript can run.

Friendly Error system

Clarence provides a friendly and adaptive error-system, that detects bugs before evaluating the program. It is a proof-of-concept!

Clarence on Neural networks

Tuesday, May 19, 2020

Since Clarence runs on a self-hosted virtual machine, we realised the it was a great idea. Now we came up with a new crazy idea.: Clarence will run on neural networks.

However, working with neural networks can be a difficult experience. Thankfully, the Clarence language supports Metaprogramming we are able to implement memory-based neurons that build an own cloud-driven runtime-network.

This article shows the conceptual idea on porting clarence's runtime to neural networks...

Motivation

In May 2020 Timo Sarkar, lead designer of Clarence, established an approach on developing a new compilation method. It is about running and hosting a vm/runtime on stack<->memory based neural-networks. "This is probably the biggest challenge we faced.
